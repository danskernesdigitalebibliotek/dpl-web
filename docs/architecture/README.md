# Cross-project architecture decision records

ADRs for decisions that span more than one sub-project (`cms/`, `go/`,
`react/`, `design-system/`, `packages/service-layer/`). Decisions
scoped to a single sub-project live under that project's own
`docs/<project>/architecture/` directory.

We follow [the format suggested by Michael Nygaard](https://cognitect.com/blog/2011/11/15/documenting-architecture-decisions)
containing the sections: *Title*, *Context*, *Decision* and *Consequences*,
plus an added *Alternatives considered* section. We do not use the
*Status* section — if an ADR is merged into the main branch it is by
default accepted.
